# Water Timer — 설계 문서

작성일: 2026-04-22
작성자: Claude (브레인스토밍 세션 기반)
상태: 초안 → 사용자 검토 대기

---

## 1. 한 줄 요약

Windows에서 백그라운드로 돌면서, 설정된 간격마다 화면 구석에 **이미지 + 메시지 + '물 마셨음' 버튼**이 있는 작은 팝업을 띄워 물 섭취를 유도하는 개인용 데스크톱 앱.

## 2. 배경과 목적

- 사용자(비전공자)가 본인 또는 지인에게 "물 마시기 리마인더"를 선물·개인용으로 쓰기 위해 기획.
- 기존 Windows 기본 알림은 위치·이미지·디자인 커스터마이징이 제한적이어서, 전용 앱이 필요.
- 핵심 가치: **따뜻하고 조용한 리마인더**. 업무를 방해하지 않되 존재감은 있어야 함.

## 3. 대상 사용자

- 주 사용자: 본인 및 본인이 선물하는 대상(가족, 친구, 연인 등).
- 기술 수준: 비전공자 포함. `.exe` 실행 한 번으로 바로 쓸 수 있어야 함.
- 모든 코드·버그 수정·업그레이드는 Claude가 담당. 사용자는 코드를 직접 다루지 않음.

## 4. 기능 요구사항

### 4.1 알림 팝업
- 일정 간격마다 화면 구석(사용자 지정 고정 위치)에 작은 창으로 등장.
- 구성: 왼쪽 이미지 + 오른쪽 메시지 + `물 마셨음` 버튼 + `×` 닫기.
- 테두리 없는 둥근 모서리, 약한 그림자, 항상 최상위(always-on-top).
- 크기: **340 × 220 픽셀** (추후 조정 가능).
- 자동 닫힘: **12초** 후 부드럽게 페이드 아웃(카운터 증가 X).
- `물 마셨음` 버튼 클릭 시 즉시 닫히고 **오늘 카운터 +1**.

### 4.2 이미지·메시지 세트
- 이미지 1장과 메시지 1줄이 **한 세트**로 묶임.
- 알림이 뜰 때마다 등록된 세트 중 **랜덤**으로 하나 선택(직전 세트와 연속 중복 방지).
- 기본 세트: 앱에 5세트가 **번들링**되어 첫 실행 시 자동 등록.
  - 기본 이미지는 구현 단계에서 사용자가 제공. 제공 전까지는 임시 물방울 아이콘으로 대체.
  - 기본 메시지 후보:
    1. "물 한 잔 어때요? 💧"
    2. "목마르지 않아도 마셔야 해요"
    3. "잠깐, 물 한 모금!"
    4. "오늘도 수고했어요. 물 한 잔 하고 가요"
    5. "탈수는 소리 없이 와요"
- 사용자는 설정 창에서 세트를 **추가·수정·삭제** 가능.

### 4.3 알림 주기
- **고정 간격**(분 단위, 기본 60분).
- **활성 시간대**(시작시각·종료시각, 기본 09:00–22:00). 이 구간 밖에서는 알림 발생 안 함.
- 구간이 자정을 넘어가는 케이스는 v1에서는 지원하지 않음(시작 < 종료 조건).

### 4.4 기록
- 오늘 `물 마셨음` 버튼을 누른 횟수만 기록.
- 날짜가 바뀌면 자동으로 0으로 리셋(다음 알림 혹은 설정 창 오픈 시점에 판정).
- 통계·그래프·히스토리는 **스코프 밖**.

### 4.5 시스템 트레이
- 앱은 창 없이 트레이 아이콘(물방울)으로만 상주.
- 우클릭 메뉴:
  - `오늘 N번 마심` (읽기 전용 카운터 표시)
  - `지금 바로 알림 보기` (테스트용)
  - `일시정지 / 재개`
  - `설정 열기`
  - `종료`
- 더블클릭: 설정 창 열기.

### 4.6 설정 창 (탭 구성)
1. **알림** — 간격(분), 활성 시간대, 팝업 위치(오른쪽 아래/오른쪽 위/왼쪽 아래/왼쪽 위/중앙), 자동 닫힘 초.
2. **이미지 & 메시지** — 세트 목록(추가/수정/삭제). 이미지 파일 선택 다이얼로그 + 메시지 텍스트 입력.
3. **기록** — 오늘 마신 횟수 표시, 수동 초기화 버튼.
4. **일반** — Windows 시작 시 자동 실행 토글(기본 ON), 버전 정보.

### 4.7 자동 실행
- Windows 레지스트리 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run`에 자기 자신 경로 등록/해제.
- 설정 토글과 실제 레지스트리 상태는 앱 시작 시 동기화.

### 4.8 일시정지
- 트레이 메뉴에서 일시정지 → 모든 알림 중단, 타이머 멈춤.
- 재개 → 재개 시점부터 간격이 새로 카운트(= 재개 즉시 알림이 뜨는 상황 방지).

## 5. 비기능 요구사항

- **플랫폼**: Windows 10/11 (64-bit).
- **설치**: PyInstaller로 빌드된 단일 `.exe`. 별도 설치 프로그램 없음. 실행 시 자동으로 사용자 데이터 폴더 생성.
- **한국어 지원**: UI 전체 한국어. 폰트는 Windows 기본(맑은 고딕).
- **시작 시간**: 실행 후 3초 이내 트레이 아이콘 표시.
- **메모리**: 상시 사용량 100MB 이하 목표.

## 6. 기술 스택

- 언어: **Python 3.11+**
- GUI: **PySide6** (Qt 6 바인딩, LGPL)
- 타이머: `QTimer`
- 자동 실행: `winreg` (Python 표준 라이브러리)
- 설정 저장: `json` (Python 표준 라이브러리)
- 테스트: `pytest`
- 빌드: `PyInstaller` (`--onefile --windowed --icon`)

## 8. 데이터 저장

- 위치: `%APPDATA%\WaterTimer\`
- 파일 2개:
  - `config.json` — 모든 설정(간격, 활성시간, 위치, 세트 목록, 자동실행 여부, 자동닫힘 초)
  - `today.json` — `{ "date": "2026-04-22", "count": 5 }`
- 파일 손상 시: 기본값으로 자동 복구. 손상 원본은 `.bak`으로 백업.
- 이미지 파일은 사용자가 등록한 경로를 그대로 저장(앱이 이미지를 복사해 오지 않음). 파일 누락 시 해당 세트만 임시 아이콘으로 대체하고 설정 창에서 경고 표시.

## 9. 동작 흐름

```
[시작]
  → config.json 로드 (없으면 기본값 + 번들 세트로 생성)
  → 자동실행 설정과 레지스트리 동기화
  → 트레이 아이콘 등록
  → scheduler 시작 (1분 tick)

[scheduler tick — 매 분]
  if 일시정지: skip
  if 현재 시각이 활성시간대 밖: skip
  if (현재시각 - 마지막 알림 시각) < 간격: skip
  → 세트 랜덤 선택
  → popup 띄움
  → 마지막 알림 시각 갱신 (config.json에 저장)

[popup 닫힘]
  if "물 마셨음" 클릭: today.json의 count +1 (날짜 다르면 리셋 후 1)
  if 자동 닫힘: 카운터 변화 없음
```

## 10. 엣지 케이스

- **절전·복귀**: tick에서 "마지막 알림 이후 경과 시간"만 보므로 복귀 시 최대 1번만 알림이 뜸(몰아치지 않음).
- **자정 전환**: tick 시점에 today.json의 date가 오늘과 다르면 count를 0으로 리셋한 뒤 판정.
- **동일 세트 연속 중복**: 직전에 쓴 세트 ID를 메모리에 보관, 다음 선택에서 제외(세트가 1개뿐이면 그냥 반복).
- **세트 0개**: 설정에서 모두 지웠을 경우 알림을 띄우지 않고 트레이 아이콘 tooltip으로 경고.
- **이미지 경로 유효성**: 팝업 시점에 실제 파일 존재 여부 확인. 없으면 임시 아이콘 fallback.
- **팝업 중 새 알림 발생**: 이전 팝업이 아직 떠 있으면 새 알림은 스킵(중복 방지).
- **다중 인스턴스 방지**: 앱은 동시에 한 개만 실행되어야 함. 뮤텍스(Windows named mutex 또는 lock 파일)로 중복 실행 감지, 이미 실행 중이면 기존 인스턴스의 설정 창을 띄우고 새 인스턴스는 종료.

## 11. 에러 처리 원칙

- 사용자 눈에 보이는 에러는 한국어로 짧게. 코드나 스택트레이스는 노출하지 않음.
- 치명적 에러는 사용자 폴더의 `error.log`에 기록(디버깅용).
- 설정 파일 손상·이미지 누락은 자동 복구 가능하므로 조용히 처리 + 설정 창에서 표시.

## 12. 테스트 전략

- **유닛 테스트(pytest)**:
  - `scheduler`의 시각 판정 로직(활성시간대, 간격 경과, 날짜 전환).
  - `settings`의 설정 로드·저장·손상 복구.
- **수동 테스트**: 트레이 메뉴의 `지금 바로 알림 보기`로 팝업 외형·위치·동작 확인.
- **E2E 자동화는 스코프 밖**(PySide6 GUI 테스트는 비용 대비 효용 낮음).

## 13. 배포

- PyInstaller 명령: `pyinstaller --onefile --windowed --icon=assets/icon.ico --add-data "assets/bundled;assets/bundled" src/app.py`
- 산출물: `dist/WaterTimer.exe` 단일 파일.
- 사용자는 원하는 폴더에 두고 더블클릭하면 끝. 첫 실행 시 %APPDATA%\WaterTimer\ 자동 생성.

## 14. v1 스코프에서 제외한 것

명시적으로 **만들지 않음**(YAGNI 원칙):

- 섭취량(ml) 입력 및 목표량 대비 달성률
- 일별/주별/월별 통계 및 그래프
- 데이터 내보내기(CSV 등)
- 클라우드 동기화·멀티 디바이스
- 소리 알림, 음성 메시지
- 다국어(한국어만)
- macOS/Linux 지원
- 자동 업데이트 기능
- 사용자 계정·로그인

필요해지면 별도 스펙으로 v2에서 다룸.
